# 基于 Unity 的气象观测科普平台  

一个基于 **Unity** 的沉浸式 **3D 气象观测科普平台**，通过互动体验与虚拟场景，提升公众对**气象科学**以及**防灾减灾**的认知水平。

---

## 1. 项目背景

### 为什么要做这个项目？

传统气象科普主要依赖文字和图片，形式单一、互动性弱，难以激发公众，尤其是青少年，对气象观测原理和气象灾害防范的兴趣。在**教育信息化**与**数字科普**快速发展的背景下，亟需一种更具**沉浸感**和**参与感**的科普方式。

### 解决了什么实际问题？

- 气象观测知识抽象、理解门槛高  
- 传统科普平台交互性弱、体验感不足  
- 科普内容更新维护成本高、扩展性差  

### 面向的用户是谁？

- 中小学及高校学生  
- 社会公众与气象科普参观者  
- 气象相关从业者及爱好者  
- 科普场馆、教育机构  

---

## 2. 项目目标

- 构建一个**沉浸式、可交互的 3D 气象观测科普平台**
- 通过虚拟展馆、天气模拟与互动问答提升学习兴趣
- 实现科普内容的数据化管理，支持灵活扩展与维护
- 探索气象科普教育的**数字化**与**游戏化**表达方式

### 成功衡量标准

- 核心功能完整、稳定运行（展馆 / 观测 / 问答 / 管理）
- 用户可独立完成学习、体验与互动流程
- 管理员可无代码维护科普内容与题库

## 4. 系统设计 / 架构

### 整体架构说明

系统采用 **C/S 架构 + MVC 设计模式**，在 Unity 客户端中实现多场景逻辑，数据通过 **SQLite 本地数据库**进行统一管理。

### 关键模块职责

- **登录与权限模块**：用户 / 管理员身份控制  
- **系统设置模块**：用户偏好与全局配置  
- **数据维护模块**：展品、题库、用户数据管理  
- **交互模块**：天气模拟、展品交互、问答系统  

### 数据流说明

用户操作 → UI 层 → Controller

↓

Model  

↓  

JSON ↔ SQLite 数据读写  

↓  

数据变更反馈至 UI

## 5. 核心功能

### 5.1 虚拟主要展馆

**功能描述**  
提供可自由漫游的 3D 虚拟展览空间，展示气象观测相关展品。

**使用场景**  
用户进入展馆，自由浏览气象仪器、模型、视频及科普内容。

**实现方式**
- Unity 场景搭建  
- 热点交互 + 信息面板  
- 多媒体内容加载  

---

### 5.2 气象观测与天气模拟

**功能描述**  
模拟多种天气条件（晴、雨、雪等）的视觉效果。

**使用场景**  
用户直观感受不同气象条件下的环境变化。

**实现方式**
- UniStorm 天气系统  
- 参数化天气切换  
- 实时渲染反馈  

---

### 5.3 气象科普互动问答

**功能描述**  
提供气象知识选择题，支持即时反馈与得分统计。

**使用场景**  
用于学习效果检测与趣味化科普。

**实现方式**
- SQLite 题库存储  
- 随机 / 顺序出题  
- 答案校验与反馈逻辑  

---

### 5.4 管理员后台管理

**功能描述**  
管理员可维护用户、展品和题库数据。

**使用场景**  
科普内容更新与系统维护。

**实现方式**
- 管理员权限校验  
- CRUD 数据操作  
- 哈希表查重算法，保证数据一致性  

---

## 6. 技术难点与解决方案

### 6.1 数据灵活管理与一致性问题

**问题描述**  
SQLite 数据类型有限，不适合直接存储复杂结构数据。

**原因分析**  
Unity 中数据结构复杂，直接映射到数据库存在困难。

**解决方案**
- 使用 JSON 作为数据中间层  
- 数据序列化 / 反序列化后再存入数据库  

**权衡 / 取舍**  
增加了序列化成本，但显著提升了系统的扩展性与维护性。

---

### 6.2 展品展示位置冲突问题

**问题描述**  
不同展品可能被分配到相同展台位置。

**解决方案**
- 引入哈希表查重算法  
- 在数据提交阶段进行校验
